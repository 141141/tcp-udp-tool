TcpUdpTool

Version: 1.0
Autor: Daniel Nilsson

Infomation

TcpUdpTool is a tool that aims to be useful during development of network applications and lets you quickly test your network communication.

TCP Client

Allows for connection and direct communication with a TCP server. Any data that is received will be displayed in chunks as they are passed up by the network interface, since TCP is a streaming protocol this may not correspond to how messages are defined by the user protocol being used.

Note: The field IP Address can be either a regular IP address or a domain name which will be automatically resolved to an IP address.

TCP Server

Start a TCP server which a client can connect to. Only a single client is allowed to connect at a time, communication with the connected client is allowed.

Note: If 0 is specified as port, then a random port in the dynamic range will be used.

Note: To bind to a local IPv6 interface IPv6 must be enabled in settings.

UDP

Send and/or receive UDP packets.

Note: It's possible to send broadcasts by specifying a broadcast address as the destination IP Address field.

Note: The destination IP Address field can be either a regular IP address or a domain name which will be automatically resolved to an IP Address.

UDP Multicast

Participate in communication in UDP multicast groups. Multicast traffic is if nothing else is specified only received on the default network interface. You may select which interface to listen on in the Interface list. You can listen to all interfaces simultainiously by selecting All. The same is true when sending multicast packets.

Note: If you are both listening and sending on multiple interfaces you may receive duplicated packages.

Esape sequences

Escape sequences can be used when sending text messages in order to send special characters or mix binary data with text.

General

\0 NULL
\a Bell (alert)
\b Backspace
\f Formfeed
\n New line
\r Carriage return
\t Horizontal tab
\v Vertical tab

Example:

Send a line of text followed by a line break:

Hello World!\r\n

Unicode

Unicode characters can be sent by using the \uXXXX escape sequence. To be able to send unicode characters you must select an appropriate encoding under Settings, e.g. utf-8. XXXX should be replaced by the character's hexadecimal unicode code point.

Example:

Send the greek letter omega (Ω).

\u03A9

Binary

Binary data can be inserted into any text message by using the \xXX escape sequnce. The binary data is inserted independent of the currently selected text encoding. XX should be replaced by the hexadeciamal byte value to insert.

Example:

Send a text message with embedded binary data at the end.

IHEAD\xFE\x09

To send binary data only, it may be easier to use the hexadeciamal message type where data is entered as space separated hexadecimal byte values.
