TcpUdpTool

Version: 1.0
Autor: Daniel Nilsson

Usage

TcpUdpTool is a tool that could be useful during development of network applications.

TCP Client

Allows for connection and direct communication with a TCP server. Any data that is received will be displayed in chunks as they are passed up by the network interface, since TCP is a streaming protocol this may not correspond to how messages are defined by the user protocol being used.

Note: The field IP Address can be either a regular IP address or a domain name which will be automatically resolved to and IP address.

TCP Server

Start a TCP server which a client can connect to. Only a single client is allowed to connect at a time, communication with the connected client is allowed.

Note: If 0 is specified as port a random port in the dynamic range will be used.

Note: To bind to a local IPv6 interface IPv6 must be enabled in settings.

UDP

Send and/or receive UDP packets.

Note: It is possible to send broadcasts by specifying a broadcast address as the destination IP Address

Note: The destination IP Address field can be either a regular IP address or a domain name which will be automatically resolved to an IP Address.

UDP Multicast

Participate in communication in UDP multicast groups. Multicast traffic is if nothing else is specified only received on the default network interface. You may select which interface to listen to in the Interface list. You can listen to all interfaces simultainiously by selecting All. The same is true when sending multicast packets.
